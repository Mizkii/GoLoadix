How it works

GoLoadix beats single-connection downloads by splitting files at the byte level.

1

HEAD request — probe the server

Checks Accept-Ranges and Content-Length. If the server supports ranges, multi-part mode activates.

2

Pre-allocate disk space

The full file size is reserved on disk upfront — no fragmentation, no surprise "disk full" errors mid-download.

3

Parallel chunk goroutines

Each chunk sends a Range: bytes=N-M request on its own TCP connection (HTTP/2 disabled to ensure true parallelism).

4

Direct disk writes

Chunks are written directly to their byte offset — no in-memory accumulation, minimal RAM usage.

5

Sidecar checkpoint file

A .goloadix file tracks each chunk's progress. On resume, completed ranges are skipped entirely.
